Unutar "packages/ivanbaric/seo" napravi profesionalni Laravel package:

Package:
ivanbaric/seo

Repository:
https://github.com/IvanBaric/seo.git

Temeljni shared infrastructure package:
ivanbaric/corexis
https://github.com/IvanBaric/corexis.git

CILJ:
Izraditi reusable SEO package koji radi u svakoj Laravel aplikaciji i na bilo kojem Eloquent modelu koji koristi HasSeo trait.

Package mora biti:
- Laravel package, ne app-specific implementacija
- backend-first
- model-agnostic
- tenant-aware preko Corexisa
- locale-aware preko Corexisa
- config-driven
- fallback-driven
- schema-aware
- sitemap-aware
- sigurnosno ispravan
- testiran
- kompatibilan s Laravel 11 i Laravel 12
- kompatibilan s PHP 8.2+
- spreman za Packagist

VAŽNO:
SEO package MORA koristiti ivanbaric/corexis za context.

Corexis daje:
- TenantResolver
- LocaleResolver
- ActorResolver ako zatreba
- SourceResolver ako zatreba
- ActionResult ako postoji i ima smisla

SEO package NE smije:
- implementirati vlastiti tenant resolver
- implementirati vlastiti locale resolver
- hardkodirati jezike
- hardkodirati team_id
- hardkodirati App\Models\User
- znati što je Team
- znati što je Velora
- biti CMS package samo za Post/Page modele
- koristiti auth() direktno
- koristiti app()->getLocale() direktno
- koristiti current_team_id() direktno
- koristiti team() direktno

Ako treba tenant, koristi:
IvanBaric\Corexis\Contracts\TenantResolver

Ako treba locale, koristi:
IvanBaric\Corexis\Contracts\LocaleResolver

Ako treba actor ili source za buduće potrebe, koristi:
IvanBaric\Corexis\Contracts\ActorResolver
IvanBaric\Corexis\Contracts\SourceResolver

Ako se stvarni API Corexis contracta razlikuje, koristi stvarni API iz lokalnog corexis packagea. Nemoj izmišljati paralelne resolvere u SEO packageu.

────────────────────────────────────────
1. PACKAGE KONCEPT
────────────────────────────────────────

Package rješava SEO za bilo koji Eloquent model:

- Post
- Page
- Vehicle
- Product
- Category
- Service
- Team
- bilo koji custom model

Model koristi:

use IvanBaric\Seo\Concerns\HasSeo;

Package mora omogućiti:

1. ručno spremljene SEO podatke u seo_meta tablici
2. automatske fallback SEO podatke iz modela
3. globalni fallback iz configa
4. Open Graph meta tagove
5. Twitter/X meta tagove
6. canonical URL
7. robots direktive
8. JSON-LD schema.org podatke
9. sitemap generiranje
10. hreflang / alternate linkove ako Corexis LocaleResolver vraća više jezika
11. tenant-aware pohranu ako je tenancy uključena u Corexisu
12. locale-aware pohranu ako je locale uključen u Corexisu

SEO fallback redoslijed:

1. ručno spremljena vrijednost iz seo_meta
2. model seoDefaults()
3. fallback atributi iz modela prema configu
4. globalni defaults iz configa
5. null

Primjer:
Ako model ima ručno unesen seo title, koristi njega.
Ako nema, koristi $model->seoDefaults()['title'].
Ako nema, pokušaj model attribute title/name/label.
Ako nema, koristi globalni site title fallback.

5. MIGRACIJA
────────────────────────────────────────

Napravi publishable migration stub:

Tablica default:
seo_meta

Polja:

id
uuid nullable unique
unique_key string(64) unique
tenant_type nullable index
tenant_id nullable index
tenant_uuid nullable index
seoable_type index
seoable_id index
seoable_uuid nullable index
locale nullable index
title nullable string
description nullable text
keywords nullable json
canonical_url nullable text
robots nullable string
og_title nullable string
og_description nullable text
og_image nullable text
og_type nullable string
twitter_title nullable string
twitter_description nullable text
twitter_image nullable text
twitter_card nullable string
schema nullable json
metadata nullable json
created_at
updated_at

Indeksi:
- unique_key unique
- seoable_type + seoable_id
- seoable_uuid
- tenant_type + tenant_id
- tenant_uuid
- locale
- updated_at

unique_key je obavezan jer nullable tenant/locale polja nisu pouzdana za composite unique index između različitih baza.

SeoUniqueKey generira hash iz:

tenant_type ili none
tenant_id ili none
seoable_type
seoable_id
locale ili __default

Primjer:
hash('sha256', "{$tenantType}|{$tenantId}|{$seoableType}|{$seoableId}|{$localeKey}")

Migracija mora koristiti:
config('seo.table.name')
config('seo.table.connection')

Ne hardkodirati connection.

7. HASSEO TRAIT
────────────────────────────────────────

Trait:
IvanBaric\Seo\Concerns\HasSeo

Dodati:

Relations:
- seoMetas(): MorphMany

Metode:
- seoMeta(?string $locale = null): ?SeoMeta
- seoMetaQuery(?string $locale = null): Builder
- getOrCreateSeoMeta(?string $locale = null): SeoMeta
- updateSeo(array $data, ?string $locale = null): SeoMeta
- seoData(?string $locale = null): SeoData
- seoDefaults(): array
- seoSchema(): array
- seoCanonicalUrl(): ?string
- seoImageUrl(): ?string

Default:
- seoDefaults() vraća []
- seoSchema() vraća []
- seoCanonicalUrl() vraća null
- seoImageUrl() vraća null
- shouldBeIndexed() vraća true

Model može overrideati bilo koju metodu.

Trait mora koristiti SeoMetaRepository / SeoManager.
Trait ne smije znati konkretne modele aplikacije.

og_title:
seo_meta.og_title
seo_meta.title

twitter_title:
seo_meta.twitter_title
seo_meta.og_title

image:
seo_meta.og_image
seoDefaults()['image']
model seoImageUrl()

config('seo.images.default')

canonical:
seo_meta.canonical_url
seoDefaults()['canonical_url']
model seoCanonicalUrl()

12. RENDERER
────────────────────────────────────────

Service:
HtmlSeoRenderer

Mora vratiti Illuminate\Support\HtmlString.

Renderira:
- title
- description
- keywords
- robots
- canonical
- Open Graph
- Twitter/X
- hreflang
- JSON-LD schema

Sigurnost:
- escape HTML output
- escape HTML attributes
- ne renderirati raw user input
- JSON-LD koristiti json_encode s JSON_UNESCAPED_SLASHES, JSON_UNESCAPED_UNICODE i sigurnosnim HEX flagovima
- ne renderirati null vrijednosti kao prazne tagove
- ne renderirati duplicate canonical tagove
- ne renderirati duplicate title tagove
- ne dopustiti javascript: URL
- ne dopustiti data: URL
- canonical i image URL moraju biti http/https absolute URL ili validan app-relative URL pretvoren u absolute URL

Renderer mora biti testiran.

────────────────────────────────────────
13. NORMALIZER I ROBOTS
────────────────────────────────────────

SeoValueNormalizer:
- trim stringova
- squish whitespace
- limit title length
- limit description length
- normalizirati keywords u array
- ukloniti prazne i duplicate keywords
- normalizirati robots string
- validirati robots directive
- pretvoriti relativni canonical URL u absolute ako config to traži

SeoRobots:
- make(string|array $directives): string
- isIndexable(string $robots): bool
- validate(string $robots): void

Podržati:
- index,follow
- noindex,follow
- index,nofollow
- noindex,nofollow
- dodatne directive iz configa

Ako model shouldBeIndexed() vrati false, default robots je noindex,nofollow osim ako ručni SEO override izričito kaže drugačije i config to dopušta.

────────────────────────────────────────
14. URL, IMAGE I SCHEMA RESOLVERI
────────────────────────────────────────

SeoUrlResolver:
- resolve(Model $model, ?string $locale = null): ?string
- alternate(Model $model, string $locale): ?string

DefaultSeoUrlResolver:
1. model seoCanonicalUrl()
2. seoDefaults()['canonical_url']
3. config sitemap.models route + route_key
4. null

SeoImageResolver:
- resolve(Model $model, ?string $locale = null): ?string

DefaultSeoImageResolver:
1. model seoImageUrl()
2. seoDefaults()['image']
3. atributi iz config fallbacks.image
4. config('seo.images.default')
5. null

SeoSchemaBuilder:
- build(Model|SeoData|array $source): array

Default schema:
- @context = https://schema.org
- @type = WebPage
- name = resolved title
- description = resolved description
- url = canonical

Ako model ima seoSchema(), mergeaj ga s default schema ako config schema.merge_model_schema = true.

Ne uvoditi hard dependency na media package.
Model može overrideati seoImageUrl().

────────────────────────────────────────
15. HREFLANG I SITEMAP
────────────────────────────────────────

HreflangGenerator:
- koristi Corexis LocaleResolver
- dohvaća available localeove
- za svaki locale koristi SeoUrlResolver::alternate()
- preskače locale ako alternate URL ne postoji
- ako include_x_default = true, koristi default locale URL kao x-default

Ne hardkodirati /hr ili /en.

SitemapGenerator:
- koristi config seo.sitemap.models
- nema znanja o app modelima
- koristi query builder
- ako model ima scopePublished ili shouldBeIndexed, koristi ih ako postoje
- koristi chunk_size iz configa
- generira XML sitemap
- koristi canonical URL resolver
- uključuje lastmod ako model ima updated_at
- uključuje hreflang alternates ako postoje
- cacheira ako cache_enabled = true

Route:
GET /sitemap.xml

Samo ako:
seo.sitemap.enabled = true
seo.sitemap.route_enabled = true
seo.routes.enabled = true

Command:
php artisan seo:generate-sitemap

Opcije:
--fresh
--write=public/sitemap.xml
--no-cache

Ako --write nije postavljen, command generira output ili warm cache.

Corexis daje tenant/locale/actor/source context.

Renderer sigurno pretvara SeoData u HTML meta tagove.
